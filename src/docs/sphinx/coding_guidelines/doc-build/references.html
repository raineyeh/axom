References and other useful resourcesÂ¶

Most of the guidelines here were gathered from the following list sources. The list contains a variety of useful resources for programming in C and C++ beyond what is presented in these guidelines.

  1. ALE3D Coding Standards & Style Guide, version 1.1, 2005.
  2. The Chromium Projects: C++ Dos and Don’ts. https://www.chromium.org/developers/coding-style/cpp-dos-and-donts
  3. Dewhurst, S., C++ Gotchas: Avoiding Common Problems in Coding and Design, Addison-Wesley, 2003.
  4. Dewhurst S., C++ Common Knowledge: Essential Intermediate Programming, Addison-Wesley, 2005.
  5. Doxygen manual, http://www.stack.nl/~dimitri/doxygen/manual/index.html
  6. Google C++ Style Guide, https://google-styleguide.googlecode.com/svn/trunk/cppguide.html
  7. Harbison, S. P. and G. L. Steele Jr., C: A Reference Manual (5th Edition), Prentice-Hall, 2002.
  8. ISO/IEC 14882:2003 C++ Programming Language Standard.
  9. ISO/IEC 14882:2011 C++ Programming Language Standard.
  10. Josuttis, N., The C++ Standard Library: A Tutorial and Reference, Second Edition, Addison-Wesley, 2012.
  11. Lakos, J., Large-Scale C++ Software Design Addison-Wesley, 1996.
  12. Meyers, S., More Effective C++: 35 New Ways to Improve Your Programs and Designs, Addison-Wesley, 1996.
  13. Meyers, S., Effective STL: 50 Specific Ways to Improve Your Use of the Standard Template Library, Addison-Wesley, 2001.
  14. Meyers, S., Effective C++: 55 Specific Ways to Improve Your Programs and Designs (3rd Edition), Addison-Wesley, 2005.
  15. Meyers, S., Effective Modern C++: 42 Specific Ways to Improve Your Use of C++11 and C++14, O’Reilly.
  16. Programming Research Ltd., High-integrity C++ Coding Standard, Version 4.0, 2013.
  17. Sutter, H. and A. Alexandrescu, C++ Coding Standards: 101 Rules, Guidelines, and Best Practices, Addison-Wesley, 2005.
